# Sibl

Sibl is a small set of primitives for building documentation inside a Next.js application. It combines an explicit manifest with ordinary MDX files without taking ownership of the framework underneath it.

<Callout title="Extracted, not invented">
  The package boundary comes from a shared implementation used by production documentation sites. Their project-specific differences remain application configuration and composition slots.
</Callout>

## One corpus, several readers

Documentation now has more than one audience. A person wants deliberate navigation and readable typography. Search needs small, ranked sections. An agent needs stable plain-text context.

Sibl derives each representation from the same MDX sources:

- a navigable React documentation surface;
- a heading-level JSON search index;
- a concise `llms.txt` map; and
- a complete `llms-full.txt` corpus.

## A visible integration

The content model begins in `sibl.config.ts`:

```ts
import { defineDocs } from "@sibl/docs";

export default defineDocs({
  title: "Acme",
  description: "Documentation for the Acme SDK.",
  navigation: [
    {
      label: "Guide",
      items: [
        {
          title: "Introduction",
          slug: "",
          source: "content/docs/index.mdx",
        },
      ],
    },
  ],
});
```

There is no directory convention pretending to be invisible configuration. Every public route maps to an explicit source file.

## The ownership boundary

| Sibl owns | Next.js and the application own |
| --- | --- |
| Documentation manifest | Routing and MDX imports |
| Shell and default MDX components | Rendering and caching strategy |
| Navigation, search, TOC, and themes | Deployment |
| Human and agent outputs | Brand and product-specific components |

That boundary keeps the package useful without turning it into another framework inside your framework.
